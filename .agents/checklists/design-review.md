# Design Review Checklist

- [ ] Visual hierarchy supports the task
- [ ] Spacing and typography are consistent
- [ ] Theme states work
- [ ] Login, Register, and NotFound use the shared public shell with no duplicate header, footer, theme control, main landmark, or copyright
- [ ] Before/after visual evidence covers light and dark themes
- [ ] Dark surfaces are neutral; light glass, inputs, text, and icons remain readable
- [ ] Glass has an opaque fallback when blur is unavailable
- [ ] Button labels describe the action and destination
- [ ] No unrelated redesign
- [ ] Mobile layouts reviewed

## Evidence Required
List files inspected, commands run, manual checks, screenshots or URLs where relevant, unresolved risks, and reviewer.
