# Testing Standards

Testing must list what was checked. Include automated checks when available, manual role/user flows, error states, permissions, responsive viewports, accessibility, security, performance, regression, and production smoke tests when deployment is requested. Never say "looks good" without evidence.

For UI changes, record routes by theme by viewport by interaction state. Minimum shell routes are `/`, `/login`, `/register`, and an invalid URL. Minimum portal coverage includes one representative admin, developer, and client route. Confirm theme persistence, one header/footer/main landmark, icon-control accessible names, 44px coarse-pointer targets, and no page-level overflow.
